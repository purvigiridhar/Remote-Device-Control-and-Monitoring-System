# Remote Device Control and Monitoring System

A socket-based IoT system that allows a central controller to remotely monitor and control Arduino devices over a secure TLS/SSL connection.

---

## Overview

This project implements a **TCP-based command and control system** where a central Python server communicates with Arduino UNO devices over an encrypted SSL connection. Commands are sent from a web-based dashboard, forwarded through the server to the Arduino client, and executed on the hardware. ACK latency is measured and displayed in real time on the dashboard.

```
Browser (Web UI) → Flask (port 8080) → Server (port 5000, TLS) → Arduino Client → Arduino UNO → LED
```

---

## Features

- **TLS/SSL encrypted communication** between server and clients
- **Token-based authentication** — devices must authenticate before receiving commands
- **Real-time web dashboard** to send commands and monitor system activity
- **Live ACK latency measurement** — round-trip time displayed on the dashboard
- **Multi-device support** — multiple Arduino clients can connect simultaneously
- **Live system log** — all commands, ACKs, and errors shown in the browser

## Setup

### 1. Generate SSL Certificate

Run this once inside the `server/` folder:

```bash
cd server
python generate_cert.py
```

This creates `server.crt` and `server.key`. Then copy `server.crt` into the `client/` folder:

```bash
copy server\server.crt client\server.crt
```

### 2. Upload Arduino Sketch

Open `client_arduino/client_arduino.ino` in the Arduino IDE and upload it to your Arduino UNO.

### 3. Find your COM Port

Open Device Manager → Ports (COM & LPT) → look for **Arduino UNO (COMX)**.

Update this line in `client/arduino_client.py`:

```python
arduino = serial.Serial('COM3', 9600)  # replace COM3 with your port
```

## Running the System

Open **3 separate terminals** and run in this order:

**Terminal 1 — Start the server:**
```bash
cd server
python server.py
```

**Terminal 2 — Connect the Arduino client:**
```bash
cd client
python arduino_client.py
```
Enter a device ID when prompted, e.g. `arduino1`.

**Terminal 3 — Start the web dashboard:**
```bash
cd client
python web_multi_ui.py
```

Open your browser and go to:
```
http://127.0.0.1:8080
```

Enter the same device ID (`arduino1`) and click **LED ON** or **LED OFF**.

## How It Works

### Authentication Flow
```
arduino_client  →  REGISTER (device_id)  →  server
arduino_client  ←  AUTH_REQUEST          ←  server
arduino_client  →  AUTH_TOKEN            →  server
arduino_client  ←  AUTH_SUCCESS          ←  server
```

### Command Flow
```
Browser → POST /command → web_multi_ui.py
       → SSL socket → server.py
       → forward to arduino_client.py
       → Serial write → Arduino UNO → LED ON/OFF
       → ACK + latency → server.py
       → POST /ack → web_multi_ui.py
       → Dashboard updates latency display
```

### SSL/TLS
- Server loads `server.crt` + `server.key` and wraps all connections with TLS
- Clients connect on the raw socket first, then perform the TLS handshake
- A self-signed certificate is used (generated by `generate_cert.py`)
